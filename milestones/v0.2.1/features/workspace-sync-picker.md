# Workspace Picker 同步入口

## 用户故事

作为已完成设备配对的用户，我希望在 Workspace Picker 中能直接选择要从对方设备同步哪些工作区，以便快速获取已有笔记。

## 依赖

- 设置页重组 + 同步 Tab（同步后可在同步 Tab 中查看状态）
- 后端工作区列表交换 API（获取对方工作区列表的数据源）

## 需求描述

当前 Workspace Picker 提供两个操作：「创建新工作区」和「打开文件夹」。v0.2.1 新增第三个入口「同步已配对设备工作区」，解决配对后「然后呢」的断点问题。

此入口在以下场景尤其重要：
1. 引导页「添加设备」路径完成配对后，进入 Workspace Picker，用户最需要的就是同步对方的工作区
2. 日常使用中，配对了新设备后想同步新工作区

## 交互设计

### Workspace Picker 改造

在现有操作按钮区域新增同步入口：

```text
┌──────────────────────────────────────────────┐
│                                              │
│             SwarmNote                        │
│                                              │
│   ┌──────────────────────────────────────┐   │
│   │  📁  创建新工作区                     │   │
│   ├──────────────────────────────────────┤   │
│   │  📂  打开文件夹                       │   │
│   ├──────────────────────────────────────┤   │
│   │  🔄  同步已配对设备工作区              │   │  ← 新增
│   └──────────────────────────────────────┘   │
│                                              │
│   最近工作区                                  │
│   ┌──────────────────────────────────────┐   │
│   │  工作笔记    ~/Documents/work-notes   │   │
│   │  个人日记    ~/Documents/diary        │   │
│   └──────────────────────────────────────┘   │
│                                              │
└──────────────────────────────────────────────┘
```

- 同步入口仅在有已配对在线设备时显示（无设备时隐藏，避免困惑）
- 如果用户刚在引导页完成配对（`pairedInOnboarding = true`），此入口高亮显示并附带引导提示

点击「同步已配对设备工作区」后弹出 Dialog：

```text
│  └──────────────────────────────────────┘    │
│                                              │

│  ┌──────────────────────────────────────┐    │
│  │  ☐  项目文档       8 篇文档          │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  同步位置：~/Documents/SwarmNote/             │
│                                   [更改...]  │
│                                              │
│  已选择本地已有的工作区将自动合并。             │
│                                              │
│           [取消]       [开始同步]              │
└──────────────────────────────────────────────┘
```

**设计要点**：

- 按设备分组展示工作区列表，设备名旁显示在线状态
- 每个工作区显示名称和文档数量，帮助用户识别
- 已在本地存在（UUID 匹配）的工作区标记为「已同步」并置灰，不可重复选择
- 默认同步位置为系统文档目录下的 `SwarmNote/` 子目录，可更改
- 每个工作区在同步位置下以工作区名创建独立子目录

- 「后台运行」按钮关闭 Dialog，同步继续在后台进行

## 技术方案

### 前端

**WorkspacePicker 组件改造**：

- 新增「同步已配对设备工作区」按钮，仅在有已配对在线设备时渲染
- 点击弹出 `WorkspaceSyncDialog` 组件

**新增组件**：

```text
src/components/workspace/
├── WorkspacePicker.tsx          # 改造：新增同步按钮
└── WorkspaceSyncDialog.tsx      # 新增：同步工作区选择 + 进度 Dialog
```

**状态管理**：

```typescript
// 可在 workspaceStore 中扩展，或新建 syncPickerStore
interface WorkspaceSyncState {
  remoteWorkspaces: RemoteWorkspaceInfo[];  // 从后端 API 获取
  selectedIds: Set<string>;                  // 用户勾选的工作区 UUID
  syncBasePath: string;                      // 同步位置
  syncProgress: Map<string, { synced: number; total: number }>;
  phase: 'selecting' | 'syncing' | 'done';
}

interface RemoteWorkspaceInfo {
  uuid: string;
  name: string;
  docCount: number;
  peerId: string;
  peerName: string;
  isLocal: boolean;  // 本地是否已存在此工作区
}
```

**引导页联动**：

- `onboardingStore.pairedInOnboarding` 为 true 时，Workspace Picker 中的同步按钮添加视觉高亮（如脉冲动画或提示文字「你已配对设备，可以同步工作区了」）
- 进入 Workspace Picker 后重置此标记

### 后端

- 调用 `get_remote_workspaces()` 获取已配对在线设备的工作区列表
- 调用 `sync_remote_workspace(workspace_uuid, peer_id, local_path)` 启动同步
- 同步完成后调用 `open_workspace_window(local_path)` 打开工作区

## 验收标准

- [ ] Workspace Picker 在有已配对在线设备时展示「同步已配对设备工作区」入口
- [ ] 无已配对在线设备时此入口不显示
- [ ] 点击入口后弹出 Dialog，按设备分组展示远程工作区列表
- [ ] 已在本地存在的工作区标记为「已同步」并不可选择
- [ ] 可勾选多个工作区，选择同步位置
- [ ] 点击「开始同步」后显示进度，逐个同步
- [ ] 同步完成后自动打开第一个工作区
- [ ] 「后台运行」可关闭 Dialog，同步继续在后台
- [ ] 引导页配对后进入 Picker，同步入口有视觉引导
- [ ] 所有 UI 适配亮色/暗色主题
- [ ] 所有文案支持中英文国际化
- [ ] `pnpm lint:ci` 通过

## 开放问题

- 同步位置默认值：Windows 用 `~/Documents/SwarmNote/`？macOS 用 `~/Documents/SwarmNote/`？还是跟随 app data 目录？
- 工作区名称冲突：如果本地已有同名但不同 UUID 的目录，如何处理？建议自动加后缀
- 大工作区（100+ 文档）的同步进度 UI 是否需要更详细的信息（如当前文件名、传输速度）？
