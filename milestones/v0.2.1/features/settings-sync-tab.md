# 设置页重组 + 同步 Tab

## 用户故事

作为用户，我希望在设置中有一个专门的「同步」页面，让我一目了然地看到工作区同步状态、同步进度和连接信息，而不是分散在「网络」和「设备」两个页面中拼凑。

## 依赖

- 无依赖（L0，可独立开始）
- v0.2.0 网络和设备页面已有基础

## 需求描述

### Tab 结构重组

当前设置窗口 4 个 Tab：**通用 / 网络 / 设备 / 关于**

重组为：**通用 / 同步 / 设备 / 关于**

- **「网络」Tab 取消**：P2P 网络启停和状态信息合并到「同步」Tab 中
- **「同步」Tab 新增**：整合网络状态 + 工作区同步状态 + 同步进度
- **「设备」Tab 保留**：聚焦配对管理（附近设备、已配对设备、配对码）
- **「通用」「关于」不变**

### 为什么这样重组

用户的心智模型是「我的笔记同步了没有」，而不是「P2P 节点是否在运行」。将网络基础设施信息包装在「同步」视角下，更符合用户期望。设备配对是独立操作（信任管理），保留单独 Tab 合理。

**布局分区**：

   - NAT 状态

2. **工作区同步列表**（中部，主体区域）
   - 列出当前设备已打开/已同步的所有工作区
   - 每个工作区显示：名称、同步状态、同步到的设备数、最后同步时间
   - 同步中的工作区显示进度条
   - 离线待同步的显示待同步文档数

3. **设置项**（底部）
   - 开机自动启动网络 Toggle

## 技术方案

### 前端

**路由变更**：

```text
src/routes/settings/
├── _layout.tsx          # SideNav 更新：网络 → 同步
├── general.tsx          # 不变
├── network.tsx          # 删除（内容合并到 sync.tsx）
├── sync.tsx             # 新增：同步 Tab 页面
├── devices.tsx          # 不变
└── about.tsx            # 不变
```

**SideNav 更新**：

```typescript
const navItems = [
  { path: '/settings/general', label: t`通用`, icon: Settings },
  { path: '/settings/sync',    label: t`同步`, icon: RefreshCw },  // 替换原 network
  { path: '/settings/devices', label: t`设备`, icon: Monitor },
  { path: '/settings/about',   label: t`关于`, icon: Info },
];
```

**新增组件**：

```text
src/routes/settings/sync.tsx         # 同步页面主体
src/components/settings/
├── NetworkStatusCard.tsx            # P2P 网络状态卡片（从 network.tsx 提取）
├── WorkspaceSyncList.tsx            # 工作区同步状态列表
└── WorkspaceSyncCard.tsx            # 单个工作区同步状态卡片
```

**状态管理**：

复用 `useNetworkStore`（网络状态）+ 扩展 `useWorkspaceStore` 或新建 `useSyncStore`：

```typescript
interface SyncStore {
  workspaceSyncStatus: Map<string, WorkspaceSyncInfo>;
  // 监听 Tauri 事件自动更新
}

interface WorkspaceSyncInfo {
  uuid: string;
  name: string;
  status: 'synced' | 'syncing' | 'pending' | 'offline';
  syncedDeviceCount: number;
  lastSyncAt: number | null;
  progress: { synced: number; total: number } | null;
}
```

### 后端

- 复用现有 Tauri 事件：`sync-progress`、`doc-sync-status-changed`
- 可能需要新增 `get_workspace_sync_status()` command 返回所有工作区的同步概况
- 网络启停 commands 不变（`start_node`、`stop_node`）
