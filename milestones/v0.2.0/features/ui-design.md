# v0.2.0 UI 设计

## 用户故事

作为用户，我希望通过直观的界面完成设备配对、查看同步状态，以便轻松管理设备间的笔记同步。

## 依赖

- 设备发现与配对（UI 需要配对相关后端 API）
- 设备状态 UI（UI 需要同步状态数据）

## 需求描述

v0.2.0 引入 P2P 同步和设备配对后，需要新增和改造一系列 UI 组件。本文档集中整理所有 UI 设计需求，方便设计时统一参考。

现有前端基础：TanStack Router + Zustand + Radix UI + TailwindCSS v4 + lucide-react 图标 + react-arborist 文件树。布局已实现 macOS/Win/Linux 平台适配。

## 设计风格

沿用 `dev-notes/design/10-ui-requirements.md` 中的定义：
- 深色主题为主，支持浅色/跟随系统切换
- 参考 Obsidian / VS Code 的布局风格
- 简洁克制，面向技术用户和知识工作者

## 二、现有组件改造

### 5. 文件树同步状态图标

在 `FileTreeNode` 中每篇文档名称右侧（或左侧图标位）显示同步状态：

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| 已同步 | `Check` (lucide) | 绿色 | 已与已连接设备同步到最新 |
| 同步中 | `RefreshCw` (lucide) | 蓝色，旋转动画 | 正在接收/发送 updates |
| 待同步 | `Clock` (lucide) | 灰色 | 全量同步队列中，尚未轮到 |
| 仅本地 | `Pencil` (lucide) | 橙色 | 有本地修改，未连接设备 |
| 无状态 | 不显示 | — | 未连接任何设备时不显示同步图标 |

**设计要点：**
- 图标要小而不干扰（12px / opacity 较低），hover 时显示 tooltip 说明
- 全量同步时大量文件显示 `⏳`，完成后逐个变为 `✅`，视觉上有"逐渐追平"的感觉
- 未连接任何设备时不显示同步图标，避免信息噪音

### 6. 状态栏（StatusBar）

现有 `StatusBar` 组件存在但未使用，需要激活并扩展。位于编辑区底部。

```
┌─────────────────────────────────────────────────┐
│ 2 设备已连接 ·  ✓ 已同步  ·  同步中 40/100 篇   │
└─────────────────────────────────────────────────┘
```

从左到右：
- **设备连接数**：`N 设备已连接` / `未连接`（点击跳转设备管理页面）
- **当前文档同步状态**：`✓ 已同步` / `⟳ 同步中` / `○ 离线待同步`
- **全量同步进度**（仅全量同步期间显示）：`同步中 40/100 篇`

**设计要点：**
- 文字要小而不突兀（12px、低对比度）
- 点击设备连接数可展开 Popover 显示已连接设备列表
- 全量同步完成后进度消失，只保留连接数和当前文档状态

### 7. 编辑区同步状态指示

在编辑器标题栏区域（文档标题旁），显示当前文档的同步状态小图标：

- `✓` 绿色 — 已同步
- `⟳` 蓝色旋转 — 同步中
- `○` 灰色 — 离线待同步

hover 显示 tooltip：如"已同步到 2 台设备" / "正在同步..." / "未连接任何设备"。

## 三、新增 Zustand Stores

### deviceStore

```typescript
interface DeviceStore {
  nearbyDevices: DeviceInfo[];      // mDNS 发现的未配对设备
  pairedDevices: PairedDeviceInfo[]; // 已配对设备（在线+离线）
  // 监听 Tauri 事件更新
}
```

### pairingStore

```typescript
type PairingPhase =
  | { phase: "idle" }
  | { phase: "generating"; codeInfo: PairingCodeInfo }
  | { phase: "inputting" }
  | { phase: "searching"; code: string }
  | { phase: "found"; device: DeviceInfo }
  | { phase: "requesting"; peerId: string }
  | { phase: "waiting-confirmation"; peerId: string }
  | { phase: "success"; device: PairedDeviceInfo }
  | { phase: "error"; message: string };

interface PairingStore {
  phase: PairingPhase;
  // 流程控制方法
}
```

### syncStatusStore

```typescript
interface SyncStatusStore {
  docSyncMap: Map<string, DocSyncStatus>;   // 文件树文档级状态

  currentDocStatus: DocSyncStatus;          // 当前打开文档的状态
  connectedDeviceCount: number;             // 已连接设备数
  // 监听 Tauri 事件更新
```

## 验收标准

### 设备管理
- [ ] 设备管理页面正确展示已配对和附近设备两个列表
- [ ] 设备卡片根据 OS 显示对应图标，区分在线/离线状态
- [ ] 附近设备列表实时更新（设备出现/消失）

### 配对流程
- [ ] Direct 配对：发起 → 等待 → 成功/拒绝 全流程 UI 完整
- [ ] Code 配对：生成码 → 展示倒计时 → 等待连接 全流程 UI 完整
- [ ] Code 配对：输入码 → 查找 → 展示设备信息 → 确认 全流程 UI 完整
- [ ] 收到配对请求时弹窗提示，显示对方设备信息
- [ ] 取消配对有二次确认弹窗

### 同步状态
- [ ] 文件树每篇文档显示同步状态图标，hover 有 tooltip
- [ ] 状态栏显示设备连接数和当前文档同步状态
- [ ] 全量同步时显示进度，完成后进度消失
- [ ] 编辑器标题区显示当前文档同步状态

### 整体
- [ ] 所有新增 UI 适配深色/浅色主题
- [ ] 所有新增 UI 适配 macOS/Windows/Linux 布局差异

## 开放问题

- 设备管理是独立页面还是设置弹窗内的 Tab？（建议先放设置弹窗内，后续内容多了再独立）
- 文件树同步图标的位置：文件名左侧（替换文件图标）还是右侧（额外信息）？
- 是否需要同步活动日志（"笔记X 已同步到 MacBook Pro"）？
- 配对请求弹窗是否需要系统级通知（应用不在前台时）？
