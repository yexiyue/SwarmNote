# 设备状态 UI

## 用户故事

作为用户，我希望能看到当前有哪些设备已连接、笔记同步状态如何，以便了解数据是否已在设备间同步。

## 依赖

- yjs CRDT 同步（需要连接和同步状态数据）
- 离线合并（需要离线/重连状态）

## 需求描述

在应用界面中展示 P2P 连接和同步状态信息，让用户直观了解设备间的同步情况。

## 交互设计

### 状态栏（全局）

位于底部或标题栏区域：
- 已连接设备数量（如 "2 设备已连接"）
- 点击展开已连接设备列表（设备名 + PeerId 缩写）
- 全局同步进度（全量同步时显示："同步中 40/100 篇"）

### 当前文档同步状态（编辑区域附近）

- 已同步 ✓ — 所有本地编辑已同步到已连接设备
- 同步中 ↻ — 正在发送/接收 updates
- 离线待同步 ○ — 未连接任何设备，有待同步内容

### 文件树文档级同步标记

文件树中每篇笔记显示同步状态图标，让用户知道哪些文档已同步、哪些还在排队：

- ✅ **已同步** — 已与已连接设备同步到最新
- 🔄 **同步中** — 正在接收/发送该文档的 updates
- ⏳ **待同步** — 全量同步队列中，尚未轮到
- ✏️ **仅本地** — 有本地修改，未连接任何设备

这在全量同步中断后尤其重要——用户可以直观看到哪些文档可能不是最新的。

## 技术方案

### 前端

- Zustand store 扩展：
  - `connectedPeers: PeerInfo[]` — 已连接设备列表
  - `syncStatus: 'synced' | 'syncing' | 'pending'` — 当前文档同步状态
  - `docSyncMap: Map<string, DocSyncStatus>` — 文件树文档级同步状态
  - `syncProgress: { synced: number, total: number } | null` — 全量同步进度
- 监听 Tauri 事件更新 store（`doc-sync-status-changed`, `sync-progress`）
- UI 组件：`DeviceStatusBar`, `PeerListPopover`, `DocSyncIcon`（文件树图标）

### 后端

- Tauri 事件：`peer-connected`, `peer-disconnected`, `sync-status-changed`
- 事件 payload 包含设备名和 PeerId

## 验收标准

- [ ] 状态栏正确显示已连接设备数量
- [ ] 可查看已连接设备列表（设备名）
- [ ] 当前文档同步状态实时更新（已同步/同步中/离线待同步）
- [ ] 文件树中每篇文档显示同步状态图标（已同步/同步中/待同步/仅本地）
- [ ] 全量同步时状态栏显示进度（如 "同步中 40/100 篇"）
- [ ] 设备连接/断开时 UI 及时更新

## 开放问题

- 状态栏放在底部还是标题栏区域
- 是否需要同步历史记录（最近一次同步时间）
- 文件树同步图标的视觉设计：图标/颜色点/角标？
