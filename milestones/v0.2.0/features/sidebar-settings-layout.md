# 侧边栏布局与设置窗口 UI 优化

## 用户故事

作为用户，我希望侧边栏更简洁、设置入口更易发现、工作区切换更便捷，以便在日常使用中获得更高效的操作体验。

## 依赖

- 无硬性依赖，基于现有 UI 组件改造

## 需求描述

当前侧边栏底部区域承载过多功能（工作区切换、网络状态、设备信息、主题/语言切换、设置入口），导致视觉拥挤。设置窗口缺少标准标题栏控制，设备管理页缺少本机信息。本需求通过重新组织布局，让各功能回归合理位置。

## 交互设计

### 1. 标题栏区域（TitleBar）

改造后的标题栏从左到右：

```
┌──────────────────────────────────────────────────────────┐
│  🟢 Logo  [工作区名称 ▾]          　　　　　　　　　[⚙]  │
├──────────────────────────────────────────────────────────┤
```

- **Logo 右侧**：工作区名称以下拉框形式展示，点击展开最近工作区列表，可直接切换
- **右上角**：设置齿轮图标按钮，点击打开设置窗口

### 2. 侧边栏底部（Sidebar）

精简为单行网络状态指示：

```
┌──────────────────────────────────┐
│  🟢 已连接 · 2 台设备在线        │
└──────────────────────────────────┘
```

- 仅保留：状态指示点 + 状态文本 + 在线设备数
- 点击可跳转到设置窗口的网络页面
- **移除内容**：工作区切换行、设备信息行、主题循环按钮、语言切换按钮、设置按钮

### 3. 设置窗口标题栏（Settings Window）

```
┌──────────────────────────────────────────────────────────┐
│  SwarmNote 设置                            [─] [□] [✕]  │
├──────────────────────────────────────────────────────────┤
│  通用  │                                                 │
│  网络  │           (设置内容区)                            │
│  设备  │                                                 │
│  关于  │                                                 │
└──────────────────────────────────────────────────────────┘
```

- 顶部添加与主窗口一致的系统操作按钮（最小化/关闭等）
- 复用 `TitleBar` 组件或采用相同的窗口装饰控制逻辑
- 平台适配：macOS 使用交通灯样式，Windows/Linux 使用自定义标题栏按钮

### 4. 设备管理页「我的设备」卡片

在设备管理页最顶部新增本机信息卡片，区分于已配对/附近设备：

```
┌──────────────────────────────────────────────────────────┐
│  我的设备                                                │
│  ┌────────────────────────────────────────────────────┐  │
│  │  🖥  MacBook Pro                                   │  │
│  │     macOS · arm64                                  │  │
│  │     Peer ID: 12D3KooW...xYz                       │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  已配对设备 (2)                                          │
│  ...                                                     │
└──────────────────────────────────────────────────────────┘
```

- 参考 SwarmDrop 的设备卡片设计风格
- 展示：设备名、操作系统、架构、Peer ID（可复制）
- 视觉上与下方列表区分（如使用 accent 边框或背景色）

### 5. 网络启停 UI 重设计

当前网络设置页使用简单的 Button + Badge 组合，改为更直观的设计：

```
┌──────────────────────────────────────────────────────────┐
│  P2P 网络                                                │
│  ┌────────────────────────────────────────────────────┐  │
│  │  ● 运行中                              [关闭网络]  │  │
│  │  Peer ID: 12D3KooW...xYz    NAT: 公网可达          │  │
│  │  已连接 2 台设备                                    │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  ○ 开机自动启动                                  [    ]  │
└──────────────────────────────────────────────────────────┘
```

- 将状态信息（Peer ID、NAT、连接数）整合到一个状态卡片中
- 启停按钮使用更明确的文案（"启动网络" / "关闭网络"），而非通用的 Start/Stop
- 停止状态时卡片变为灰色调，按钮变为"启动网络"

## 验收标准

- [ ] 标题栏 Logo 旁以下拉框展示当前工作区名称，可切换最近工作区
- [ ] 设置按钮位于标题栏右上角，点击可打开设置窗口
- [ ] 侧边栏底部仅保留一行网络状态（状态点 + 文本 + 设备数）
- [ ] 主题/语言快捷切换已从侧边栏移除（功能保留在设置 > 通用中）
- [ ] 设置窗口顶部有与主窗口一致的系统操作按钮，平台适配正确
- [ ] 设备管理页顶部展示「我的设备」卡片（设备名、OS、Peer ID）
- [ ] 网络启停交互重新设计，状态信息集中展示，按钮文案明确
- [ ] 所有改动适配亮色/暗色主题
- [ ] 所有改动适配 macOS / Windows / Linux 平台布局
- [ ] 所有新增文案支持中英文国际化
- [ ] `pnpm lint:ci` 通过
