# 离线合并

## 用户故事

作为用户，我希望断开网络后继续编辑笔记，重新连接后自动合并所有离线期间的编辑，不产生冲突。

## 依赖

- yjs CRDT 同步（基于全量同步机制实现）

## 需求描述

支持完整的离线编辑场景：设备离线后继续编辑，所有 yjs updates 本地持久化。重新连接后自动触发全量同步，利用 yjs CRDT 特性无冲突合并双方的离线编辑。

## 技术方案

### 后端

- 离线期间：yjs updates 正常存入 SQLite，.md 文件正常保存
- 重连检测：`NodeEvent::PeerConnected` → 对每个已打开文档触发全量同步
- 全量同步流程：交换 state_vector → 互发缺失 updates → 合并 → 写回 .md
- 合并后通知前端刷新编辑器内容

### 前端

- 监听重连事件 → 等待同步完成 → 刷新 Y.Doc → 编辑器自动更新
- 同步状态指示：离线待同步 → 同步中 → 已同步

## 验收标准

- [ ] 关闭 B → A 编辑多条内容 → 重开 B → B 自动追上所有编辑
- [ ] A 和 B 同时离线编辑同一文档 → 重连后 CRDT 自动合并
- [ ] 合并后两端内容一致
- [ ] 合并后 .md 文件正确更新
- [ ] 离线期间的编辑不会丢失

## 开放问题

- 长时间离线后 yjs updates 积累较多，全量同步性能如何
- 是否需要同步进度指示（大文档同步可能需要几秒）
