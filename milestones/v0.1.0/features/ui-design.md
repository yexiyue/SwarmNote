# UI 设计稿

## 用户故事

作为开发者，我需要明确的视觉设计参考，以便实现时有清晰的目标，避免边写代码边摸索 UI。

## 需求描述

为 v0.1.0 的所有用户界面制作设计稿，覆盖 Onboarding 流程、主界面布局、文件树、编辑器等核心页面。设计稿存放在 `milestones/v0.1.0/design/` 目录下。

## 设计范围

### 需要设计的页面/组件

| 页面 | 说明 | 优先级 |
|------|------|--------|
| 主界面布局 | 侧边栏 + 编辑区 + 状态栏的整体布局 | P0 |
| 文件树 | 文件夹/文件节点、展开/折叠、右键菜单、行内重命名 | P0 |
| 编辑器区域 | BlockNote 编辑器 + 文档标题 + 状态栏 | P0 |
| Onboarding 流程 | 欢迎页、工作区选择、设备名设置、完成页（4 步） | P0 |
| 空状态 | 无笔记时的引导提示 | P1 |
| 确认对话框 | 删除确认等通用对话框样式 | P1 |

### 设计风格参考

- 暗色主题为主，参考 Obsidian / VS Code 风格
- 中性色调，低对比度背景，高可读性文本
- 参考 `dev-notes/design/10-ui-requirements.md` 中的详细描述
