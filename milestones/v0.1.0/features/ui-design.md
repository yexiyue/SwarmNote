# UI 设计稿

## 用户故事

作为开发者，我需要明确的视觉设计参考，以便实现时有清晰的目标，避免边写代码边摸索 UI。

## 需求描述

为 v0.1.0 的所有用户界面制作设计稿，覆盖 Onboarding 流程、主界面布局、文件树、编辑器等核心页面。设计稿存放在 `milestones/v0.1.0/design/` 目录下。

## 设计范围

只做桌面端，不做移动端。

### 设计页面清单

| 页面 | 说明 | 亮色 | 暗色 |
|------|------|------|------|
| 主界面 | 标题栏 + 侧边栏 + 编辑器 + 状态栏 | ✅ | ✅ |
| 空状态 | 无笔记时的引导提示（新建按钮 + Ctrl+N 提示） | ✅ | ✅ |
| Onboarding 1 | 欢迎页（Logo + 特性列表 + 开始按钮） | ✅ | ✅ |
| Onboarding 2 | 选择工作区目录（路径选择器 + 浏览按钮） | ✅ | ✅ |
| Onboarding 3 | 设备名称设置（输入框 + 默认主机名） | ✅ | ✅ |
| Onboarding 4 | 完成页（配置摘要：工作区/设备名/PeerId） | ✅ | ✅ |
| 删除确认对话框 | 模态遮罩 + 对话框（取消/删除按钮） | ✅ | ✅ |

共 14 个屏幕（7 亮色 + 7 暗色）。

### 设计风格

- **主色调**：靛蓝 Indigo #4F46E5（亮色）/ #818CF8（暗色）
- **基础色**：Slate 系列，通过 CSS 变量自动切换亮暗
- **主题**：亮色为默认，支持暗色主题切换
- **字体**：Inter（UI）+ JetBrains Mono（代码块）
- **组件库**：shadcn/ui，按需安装
- **参考**：Linear / Notion 风格，干净简洁

### 布局决策

- **标题栏**：左侧 Logo + 导航，右侧搜索框 (Ctrl+P) + 设置按钮 + 窗口操作按钮（最小化/最大化/关闭）
- **侧边栏**（240px）：工作区名 + 新建按钮 + 文件树 + 底部设备信息（设备名 + PeerId 缩写）
- **编辑区**：无工具栏，沉浸式编辑，文档标题直接在编辑区内
- **状态栏**：字数 + 字符数 + 保存状态
- **无右侧面板**：两栏布局（侧边栏 + 编辑区）
