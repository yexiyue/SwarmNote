# SwarmNote 开发任务拆分

## 开发策略

**最小可用 → 逐步迭代**：三个阶段依次推进，每个阶段有独立的完成标志和验证点。

## 三个阶段

| 阶段 | 目标 | 完成标志 |
|------|------|---------|
| [Phase 1：P2P 网络层](phases/phase1-networking.md) | 节点身份、多设备关联、节点发现、信任认证、NAT 穿透、文件同步 | 两台设备信任后同步图片；多设备配对关联自动同步 |
| [Phase 2：编辑器 + CRDT 协作](phases/phase2-editor.md) | CodeMirror 编辑器、yrs/yjs 文档同步、协作感知 | 两台设备实时协作编辑 Markdown，离线重连自动合并 |
| [Phase 3：端到端加密](phases/phase3-encryption.md) | E2E 加密、密钥分发、文档分享 | 文档 E2E 加密传输，通过分享码邀请协作，密钥轮换 |

## Phase 1 详细任务

### 阶段 A：基础设施

| 任务 | 内容 | 验证标准 |
|------|------|---------|
| 1.1 Swarm 初始化 | libp2p Swarm + Identify + 事件循环 | 应用启动，日志输出 PeerId |
| 1.2 密钥持久化 | Ed25519 密钥生成与加密存储 | 重启后 PeerId 不变 |
| 1.9 本地存储 | SeaORM + SQLite + Migration | 数据持久化正常 |
| 1.13 错误处理 | 错误类型定义 + tracing 日志 | 日志记录完整 |

### 阶段 B：局域网功能

| 任务 | 内容 | 验证标准 |
|------|------|---------|
| 1.3 用户身份与多设备 | 用户身份、配对码、设备关联 | 配对码关联两台设备 |
| 1.4 mDNS 发现 | 局域网节点自动发现 | 两台设备互相发现 |
| 1.5 节点信任 | 信任级别、弹窗确认、双向信任 | 信任后连接，拒绝后忽略 |
| 1.6 连接管理 | 连接建立、维护、断开、重连 | 断网后自动重连 |
| 1.7 文件同步 | Request-Response、大文件分块 | 同步 10MB 图片成功 |
| 1.8 同步状态 | 队列管理、进度追踪、冲突处理 | 显示同步进度，处理冲突 |

### 阶段 C：跨网络功能

| 任务 | 内容 | 验证标准 |
|------|------|---------|
| 1.10 Kademlia DHT | Provider Records、Peer Routing | 跨网络发现节点 |
| 1.11 NAT 穿透 | AutoNAT + Relay + DCUtR | NAT 后设备可连接 |
| 1.12 引导节点 | swarmnote-bootstrap 程序 + 部署 | 引导节点正常运行 |

### 阶段 D：前端集成

| 任务 | 内容 | 验证标准 |
|------|------|---------|
| 1.14 前端 UI | 首次引导、配对、信任、文件管理、状态栏 | 端到端流程顺畅 |

## Phase 1 里程碑

| 里程碑 | 内容 | 验证标准 |
|--------|------|---------|
| M1 | 基础设施就绪 | Swarm 启动，PeerId 持久化，数据库可用 |
| M2 | 局域网 P2P | 两台设备局域网内发现、信任、同步文件 |
| M3 | 多设备关联 | 同一用户多台设备配对关联，自动同步 |
| M4 | 跨网络 P2P | 不同网络的设备通过 DHT 发现并同步 |
| M5 | 完整 UI | 前端界面完成，端到端流程顺畅 |

## 依赖关系

```
Phase 1: P2P 网络层
  │
  ├──> Phase 2: 编辑器 + CRDT 协作
  │
  └──> Phase 3: 端到端加密（依赖 Phase 1 网络 + Phase 2 文档同步）
```

Phase 2 和 Phase 3 都依赖 Phase 1，但 Phase 2 和 Phase 3 之间有部分依赖（Phase 3 的加密需要集成到 Phase 2 的同步通道中），建议 Phase 2 先行，Phase 3 在 Phase 2 基础上叠加。
