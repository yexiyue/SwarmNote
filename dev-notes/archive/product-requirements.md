# SwarmNote 产品需求文档

## 产品定位

SwarmNote 是一款**去中心化、离线优先、开源**的个人笔记应用，支持 P2P 实时协作编辑。

定位类似"去中心化的 Obsidian"：以个人 Markdown 笔记为主，协作为辅。用户数据完全本地存储，无需任何云服务，通过 P2P 网络实现设备间和用户间的文档同步。

### 目标用户

个人笔记用户。主要自己记录，偶尔与朋友或同事共享编辑。

### 核心差异化

与 Notion、Obsidian 等产品的区别：

| 维度 | Notion | Obsidian | SwarmNote |
|------|--------|----------|-----------|
| 数据存储 | 云端 | 本地文件 | 本地 SQLite |
| 协作方式 | 中心化服务器 | 需付费 Sync | P2P 直连，免费 |
| 离线能力 | 有限 | 完整 | 完整 + CRDT 自动合并 |
| 开源 | 否 | 否 | 是 |
| 自托管 | 否 | 不需要 | 可自建引导节点 |
| 隐私 | 数据在云端 | 本地 | 本地 + E2E 加密传输 |

## 功能清单

### P0：核心功能（MVP 必须）

#### 1. Markdown 编辑器

- 所见即所得的 Markdown 编辑体验
- 支持基础语法：标题、列表、加粗、斜体、链接、引用、代码块、分隔线
- 图片插入与显示（本地图片嵌入）
- 键盘快捷键支持

#### 2. 文档管理

- 文档的创建、重命名、删除
- 平坦列表展示所有文档（MVP 阶段不做嵌套目录）
- 文档排序（按修改时间、标题）

#### 3. 本地持久化

- SQLite 存储文档内容（yrs 二进制快照）
- 文档元数据存储（标题、创建时间、最后编辑时间）
- 应用关闭后数据不丢失

#### 4. P2P 网络连接

- 局域网自动发现（mDNS）
- 手动输入节点地址连接
- 引导节点中转（支持跨网络连接）
- 连接状态展示（已连接节点数、节点列表）

#### 5. 实时协作同步

- 基于 CRDT（yrs/yjs）的无冲突实时同步
- 增量同步：GossipSub 广播文档变更
- 全量同步：新节点加入时通过 Request-Response 拉取完整文档
- 灵活的分享粒度：可分享单篇文档或整个文件夹

#### 6. 用户身份

- 本地生成密钥对作为底层身份标识
- 用户设置友好昵称和头像颜色
- 协作时显示对方身份信息

#### 7. 协作感知

- 实时显示协作者光标位置
- 显示协作者文本选区
- 在线状态展示（谁在线、谁在编辑哪篇文档）

#### 8. 端到端加密

- 文档内容 E2E 加密，仅持有密钥的节点可读取
- 基于 libp2p noise 的传输层加密
- 分享文档时通过密钥交换授权

#### 9. 完全离线可用

- 断网状态下正常创建、编辑文档
- 重新联网后 CRDT 自动合并，无需手动处理冲突
- 离线期间的编辑完整保留，不丢失

### P1：重要功能（MVP 包含，优先级次于 P0）

#### 10. 版本历史

- 查看文档的历史版本
- 按时间线浏览变更记录
- 可回退到特定历史版本

#### 11. 全文搜索

- 搜索本地所有文档内容
- 关键词高亮显示
- 搜索结果快速跳转

#### 12. 导入功能

- 导入 Markdown 文件（.md）
- 导入 Obsidian vault（批量 .md + 保留文件结构）
- 导入 Notion 导出文件（Markdown 格式）

#### 13. 导出功能

- 导出单篇文档为 Markdown 文件
- 导出单篇文档为 PDF
- 批量导出所有文档

### P2：后续迭代

#### 14. 双向链接与图谱

- `[[wikilink]]` 语法支持
- 点击链接跳转到对应文档
- 反向链接面板（哪些文档引用了当前文档）
- 文档关系图谱可视化

#### 15. 高级笔记功能

- 标签系统
- 文档模板
- 日记 / 每日笔记

#### 16. 高级文档组织

- 树形文件夹目录
- 拖拽排序
- 收藏 / 置顶

#### 17. 移动端支持

- Android / iOS 客户端（Tauri 移动端）

## 技术架构概览

```
┌──────────────────────────────────┐
│          React 前端               │
│                                  │
│  CodeMirror 6 (Markdown 编辑)    │
│    ↕ y-codemirror.next           │
│  Y.Doc (yjs)                     │
│    ↕ binary encoding             │
│  Tauri invoke / events           │
└──────────┬───────────────────────┘
           │ IPC (binary)
┌──────────▼───────────────────────┐
│        Tauri Rust 后端            │
│                                  │
│  yrs::Doc (CRDT)                 │
│  SQLite (持久化)                  │
│  E2E 加密层                       │
│    ↕                             │
│  libp2p Swarm                    │
│    ├─ GossipSub (增量同步)        │
│    ├─ Request-Response (全量同步) │
│    ├─ mDNS (局域网发现)           │
│    └─ Relay/DCUtR (NAT 穿透)     │
└──────────────────────────────────┘
```

### 技术选型

| 层面 | 选型 | 说明 |
|------|------|------|
| 应用框架 | Tauri v2 | 跨平台桌面应用，Rust 后端 + Web 前端 |
| 前端框架 | React + TypeScript | |
| 编辑器 | CodeMirror 6 | 原生 Markdown 编辑体验，y-codemirror.next 绑定 |
| CRDT | yrs (后端) / yjs (前端) | 二进制编码兼容，无需序列化转换 |
| P2P 网络 | libp2p | GossipSub + Request-Response + mDNS |
| 持久化 | SQLite (rusqlite) | 文档内容以 yrs BLOB 存储 |
| 全文搜索 | tantivy 或 SQLite FTS5 | 待定 |

### 与原技术选型的调整

原技术选型文档选择了 TipTap（富文本编辑器），但产品定位为 Markdown 编辑器后，调整为 **CodeMirror 6**：

- CodeMirror 6 原生支持 Markdown 语法高亮和编辑
- y-codemirror.next 提供 yjs 协作绑定
- 更贴近 Obsidian 的编辑体验
- TipTap 更适合富文本 / Block 编辑器场景

## 用户流程

### 首次使用

1. 安装并打开应用
2. 自动生成密钥对（用户无感）
3. 设置昵称和头像颜色
4. 进入空的文档列表，开始创建第一篇笔记

### 日常使用（个人）

1. 打开应用 → 文档列表
2. 新建或点击已有文档 → 进入编辑器
3. Markdown 编辑，自动保存
4. 通过搜索快速找到文档

### 发起协作

1. 选择文档（或文件夹）→ 点击"分享"
2. 生成分享链接 / 分享码（包含文档 ID + 加密密钥）
3. 发送给对方（通过任意通讯工具）
4. 对方在 SwarmNote 中打开分享链接
5. 自动建立 P2P 连接，开始同步
6. 双方实时看到对方的编辑和光标

### 离线 → 重连

1. 断网后继续编辑（完全正常使用）
2. 重新联网 → 自动发现已连接过的节点
3. CRDT 自动合并双方离线期间的编辑
4. 无冲突提示，无需手动操作

## 非功能需求

### 性能

- 编辑延迟 < 16ms（60fps 流畅输入）
- P2P 同步延迟 < 500ms（局域网内）
- 支持单文档 10万字以上不卡顿
- 应用启动时间 < 3s

### 安全

- 密钥对本地生成和存储，不外传
- 文档内容端到端加密
- P2P 传输层 noise 加密
- 无遥测、无数据上报

### 可靠性

- 应用异常退出不丢失数据（WAL 模式 SQLite）
- CRDT 保证最终一致性，不会出现数据冲突丢失
- 本地数据为唯一数据源，不依赖任何远程服务

## 风险与对策

| 风险 | 对策 |
|------|------|
| CodeMirror 6 + y-codemirror.next 协作体验不如 TipTap 成熟 | 预研验证，必要时考虑 Milkdown（Markdown + ProseMirror + yjs）作为备选 |
| E2E 加密增加同步复杂度 | MVP 先实现密钥交换 + 对称加密的基础方案，不做复杂的群组密钥管理 |
| 大文档全量同步慢 | yrs 支持快照压缩，必要时分块传输 |
| NAT 穿透失败率 | 提供引导节点 / Relay 作为兜底 |
| 导入 Obsidian vault 格式差异 | 先支持纯 Markdown 导入，wikilink 等特殊语法在 P2 阶段处理 |

## MVP 开发阶段划分（建议）

### Phase 1：单机可用

- Tauri 项目脚手架
- CodeMirror 6 Markdown 编辑器集成
- 文档 CRUD + SQLite 持久化
- 基础 UI 布局（文档列表 + 编辑区）

### Phase 2：P2P 联通

- libp2p Swarm 初始化（Rust 后端）
- mDNS 局域网发现
- 手动节点连接
- yrs 文档同步（GossipSub 增量 + Request-Response 全量）
- Tauri IPC 打通前后端 yjs/yrs

### Phase 3：协作体验

- 用户身份（密钥对 + 昵称）
- 协作者光标 / 选区 / 在线状态（Awareness）
- 文档分享流程
- E2E 加密

### Phase 4：完善 MVP

- 版本历史
- 全文搜索
- 导入 / 导出
- 引导节点 + NAT 穿透
- 图片管理和 P2P 同步

## 术语表

| 术语 | 说明 |
|------|------|
| CRDT | Conflict-free Replicated Data Type，无冲突复制数据类型 |
| yrs | yjs 的 Rust 实现，用于后端文档同步 |
| yjs | JavaScript CRDT 库，用于前端编辑器协作绑定 |
| GossipSub | libp2p 的发布-订阅协议，用于广播文档增量更新 |
| Request-Response | libp2p 的请求-响应协议，用于全量文档同步 |
| mDNS | 局域网多播 DNS，用于自动发现本地网络中的节点 |
| Awareness | yjs 的用户感知协议，用于同步光标、选区、在线状态 |
| E2E | End-to-End，端到端加密 |
| NAT | Network Address Translation，网络地址转换 |
| Relay | libp2p 中继节点，用于帮助 NAT 后的节点建立连接 |
