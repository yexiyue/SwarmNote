# UI 设计需求

> 基于系统设计文档和产品愿景，整理桌面端 UI 需要设计的页面和交互。
> 当前设计稿 `dev-notes/ui-design/desktop.pen` 已过时，需要重新设计。

## 设计风格

- 深色主题为主，支持浅色/跟随系统切换
- 参考 Obsidian / VS Code 的布局风格（侧边栏 + 编辑区）
- shadcn/ui 组件库（已在 .pen 中有设计系统组件）
- 简洁克制，面向技术用户和知识工作者

## 页面清单

### P0：核心体验（MVP 必须）

#### 1. 主编辑界面

整个应用的核心页面，日常使用频率最高。

```
┌──────────────────────────────────────────────┐
│ 标题栏：SwarmNote    同步状态    搜索    设置 │
├──────────┬───────────────────────────────────┤
│ 文件树   │ 编辑区                             │
│          │                                   │
│ 工作区名 │ 文档标题（可编辑）                  │
│ ├─ 📁目录│                                   │
│ │  ├─ 📄 │ BlockNote 编辑器                   │
│ │  └─ 📄 │ （Markdown 渲染）                  │
│ ├─ 📄    │                                   │
│ └─ 📄    │                                   │
│          │                                   │
│ ──────── │                                   │
│ + 新建   │                                   │
│          │ 状态栏：字数 · 同步状态 · 最后保存  │
└──────────┴───────────────────────────────────┘
```

**文件树侧边栏：**
- 树状结构，直接映射文件系统目录
- 文件夹可展开/折叠
- .md 文件显示文档图标 + 标题（不显示 .md 后缀）
- 同名资源目录（如 `学习笔记/`）不在文件树中显示（隐藏实现细节）
- 右键菜单：新建文件、新建文件夹、重命名、删除、分享
- 拖拽排序/移动文件
- 文件同步状态图标：✓ 已同步 / ⟳ 同步中 / ⚠ 有冲突

**编辑区：**
- BlockNote 编辑器，block-based
- 支持 Markdown 快捷键（`#` 标题、`-` 列表、`>` 引用等）
- 图片插入：拖拽/粘贴/按钮 → 保存到同名资源目录
- 文档标题 = 文件名（编辑标题即重命名文件）

**状态栏：**
- 当前文档字数/字符数
- 同步状态（已同步 / 正在同步到 X 台设备 / 离线）
- 最后保存时间
- 在线协作者头像（L1 实时协作时显示）

#### 2. 工作区初始化/选择

首次启动或切换工作区时的流程。

**初始化流程：**
```
首次启动
→ "选择工作区目录"（文件夹选择器）
→ 在选中目录下创建 .swarmnote/
→ 进入主编辑界面
```

**工作区切换：**
- 设置中可管理多个工作区
- 快捷切换（下拉菜单或 Ctrl+O）
- 显示每个工作区的路径、文档数量、最后打开时间

#### 3. Onboarding 欢迎流程

已有设计，需要调整：

- **Step 1**：欢迎页（保留现有设计）
- **Step 2**：选择/创建工作区目录（替代原来的"个人资料"步骤）
- **Step 3**：设置设备名称（用于配对时显示）
- **Step 4**：完成，进入主界面

### P1：同步与设备（MVP 后第一批）

#### 4. 设备管理页面

已有设计基础，需要更新。

- 已配对设备列表（设备名、系统、在线状态、最后在线时间）
- 每个设备可查看：共享了哪些工作区/文件夹、角色
- "添加设备"按钮 → 显示 6 位配对码
- "输入配对码"入口

#### 5. 同步状态面板

可作为侧边栏的一个 tab 或独立面板。

- 当前工作区的同步概览：
  - X 个文件已同步
  - X 个文件正在同步
  - X 个文件有冲突
- 最近同步活动日志（"学习笔记.md 已同步到 MacBook Pro"）
- 按设备分组查看同步状态

#### 6. 冲突解决界面

当检测到冲突时（两端都修改了同一文件 / 外部编辑冲突）。

**外部编辑提示（Toast/Banner）：**
```
┌──────────────────────────────────────────┐
│ ⚠ "学习笔记.md" 已被外部程序修改         │
│                                          │
│ [重新加载]  [忽略]  [查看差异]            │
└──────────────────────────────────────────┘
```

**冲突文件提示：**
- 文件树中冲突文件标记 ⚠ 图标
- 点击后显示：本地版本 vs 远端版本
- 操作：保留本地 / 保留远端 / 手动合并
- .conflict 文件在文件树中以特殊样式显示

## 交互细节

### 文件操作

| 操作 | 触发方式 | 行为 |
|------|---------|------|
| 新建文档 | 侧边栏 "+" / Ctrl+N | 在当前目录创建 `未命名.md`，光标定位到标题 |
| 新建文件夹 | 侧边栏右键 | 在当前目录创建文件夹 |
| 删除文档 | 右键 → 删除 | 确认对话框，同时删除同名资源目录 |
| 重命名 | 双击文件名 / F2 | 重命名 .md 文件和同名资源目录 |
| 移动 | 拖拽 | 移动 .md 文件和同名资源目录 |
| 搜索 | Ctrl+P | 模糊搜索文件名，快速打开 |
| 全文搜索 | Ctrl+Shift+F | 搜索所有 .md 文件内容 |

### 同步指示

| 状态 | 图标 | 说明 |
|------|------|------|
| 已同步 | ✓ 绿色 | 所有已连接设备都有最新版本 |
| 同步中 | ⟳ 蓝色 | 正在传输中 |
| 有冲突 | ⚠ 黄色 | 存在 .conflict 文件或合并冲突 |
| 仅本地 | — 灰色 | 未分享的私有文档 |
| 离线 | ○ 灰色 | 无网络连接 |

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+N | 新建文档 |
| Ctrl+P | 快速打开（文件搜索） |
| Ctrl+Shift+F | 全文搜索 |
| Ctrl+S | 手动保存（虽然自动保存，但保留习惯） |
| Ctrl+, | 打开设置 |
| Ctrl+B | 切换侧边栏 |
| Ctrl+Shift+E | 聚焦文件树 |

## 与现有设计稿的差异

| 现有设计 | 需要改为 |
|---------|---------|
| 内容区显示架构文档 | BlockNote 编辑器 |
| 扁平文件列表 | 树状文件系统结构 |
| 无工作区概念 | 工作区目录选择/管理 |
| 无同步状态 | 文件级同步指示器 |
| 无分享入口 | 右键菜单 + 分享对话框 |
| 无冲突提示 | 外部编辑检测 + 冲突解决 UI |
| Onboarding 无工作区步骤 | 第二步选择工作区目录 |

## 设计排期建议

1. **先设计 P0 页面**（主编辑界面 + 文件树 + 工作区初始化），这是 MVP 的核心
2. 然后 P1（同步状态 + 冲突解决），配合同步功能开发
3. 最后 P2-P3（分享 + 权限 + 完整设置），配合后续 Phase
